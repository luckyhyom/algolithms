### Direct Addressing Table (DAT)
정의: 값(Value)이 곧 주소(Index)가 되는 배열, 카운팅 배열

**Hashing 과의 관계**\
DAT는 복잡한 해시 함수와 충돌 해결 로직을 생략한, 가장 빠르고 순수한 형태의 해시 테이블이다.

**결정적인 차이를 보여주는 시나리오** \
전화번호부를 만든다고 가정해 봅시다.
1. DAT 방식: 010-0000-0000부터 010-9999-9999까지 1억 개의 칸을 미리 다 만들어 둡니다. 내 친구가 딱 10명뿐이라도 1억 칸짜리 메모리를 점유해야 합니다. (매우 비효율적)
> 검색 시간 복잡도 O(n)
2. Hashing 방식: 내 친구가 10명이면 약 20칸 정도의 배열만 준비합니다. 전화번호를 해시 함수에 넣어 0~19 사이의 숫자로 바꾸어 저장합니다. (메모리 절약)
> 검색 시간 복잡도 O(1)

DAT를 쓰세요: 데이터의 최댓값이 100만 이하이거나, 범위가 작아서 메모리가 아깝지 않을 때. (알고리즘 문제 풀이, 고정된 상태 값 저장 등)\
해싱을 쓰세요: 데이터가 문자열이거나, 숫자 범위가 수십억 대를 넘어가서 도저히 배열로 감당이 안 될 때. (DB 인덱스, 캐시 시스템, Map 자료구조, 학번, 주민번호, 등)
